此技能指导创建独具风格、生产级前端界面，避免"AI 批量感"的泛化美学。实现真实可用的代码，在审美细节和创意选择上精雕细琢。

用户提出前端需求：一个组件、页面、应用或界面。可能包含用途、受众或技术约束等背景信息。

当作为 `frontend-builder` 的审美子技能使用时，优先输出一张**设计决策卡片**：项目场景、美学方向、实体世界参照物、字体搭配、配色方案、空间策略、记忆点。后续代码与样式必须回溯到这张卡片。

## 设计思考

在编码之前，先理解场景并锁定一个**大胆**的美学方向：

- **目的**：这个界面解决什么问题？谁在使用？
- **基调**：选取一种极致风格：极致极简、繁复主义、复古未来、自然有机、奢华精致、趣味玩具风、杂志编辑风、粗野主义、装饰艺术几何、柔和粉彩、工业实用风……可选项极其丰富。以上为灵感参考，但要打造真正契合该方向的独特设计。
- **约束**：技术需求（框架、性能、无障碍）。
- **记忆点**：什么让这个设计令人难以忘怀？用户会记住哪一点？

**关键**：选定清晰的概念方向并精准执行。大胆的繁复主义与克制的极简主义同样出色——关键在于意图明确，而非强度大小。

然后实现可运行的代码（HTML/CSS/JS、React、Vue 等），要求：
- 生产级质量，功能完备
- 视觉冲击力强，令人印象深刻
- 具有清晰美学主张的统一风格
- 每个细节都经过精心打磨

## 前端美学指南

重点关注：

- **字体排印**：选择优美、独特、有趣的字体。避免 Arial、Inter 等通用字体；优先选择能提升界面美感的独特字体；出人意料、富有个性的字体搭配。将独特的展示字体与精致的正文字体配对。中文字体推荐优先使用霞鹜文楷、思源宋体、阿里妈妈方圆体等有特色的字体，避免机械地使用系统默认字体。

- **色彩与主题**：坚定地选择统一的美学调性。使用 CSS 变量保持一致性。主色鲜明、点缀色利落的配色优于畏首畏尾、均匀分配的调色板。

- **动效**：用动画强化效果和微交互。HTML 场景优先使用纯 CSS 方案；React 场景可用 Motion 库。聚焦高光时刻：一次精心编排的页面加载动画（配合 animation-delay 做逐次揭示）比零散的微交互更能制造惊喜。善用滚动触发和出人意料的悬停状态。

- **空间构成**：打破常规的布局。不对称。叠加。对角线流动。打破网格的元素。大面积留白或高密度控制。

- **背景与视觉细节**：营造氛围和层次感，而非默认使用纯色。贴合整体美学，添加场景化的效果和纹理。创造性地运用渐变网格、噪点纹理、几何图案、分层透明、戏剧化阴影、装饰性边框、自定义光标、颗粒感叠加等手法。

**绝对不要**使用泛化的 AI 美学：如过度使用的字体家族（Inter、Roboto、Arial、系统字体）、陈旧的配色套路（尤其是白底紫色渐变）、可预测的布局和组件模式、缺乏场景特色的模板式设计。

创造性解读需求，做出令人意外的选择，让设计真正为当前场景而生。每次设计都应该不同。在浅色和深色主题、不同字体、不同美学之间灵活变化。永远不要在多次生成中趋同于某些流行选择（例如 Space Grotesk）。

**重要**：实现复杂度要与美学愿景匹配。繁复主义设计需要精心编码，配合大量动画和效果。极简或精致设计则需要克制、精准，对间距、字体排印和微妙细节格外用心。优雅来源于对愿景的出色执行。

记住：你拥有非凡的创造力。跳出框架思考，全身心投入独特的视觉愿景，展示真正的设计能力。

## 中文场景特别建议

- **字体选择**：正文优先使用霞鹜文楷（LXGW WenKai）、思源黑体（Noto Sans SC）、阿里巴巴普惠体等优质中文字体。展示字体可考虑思源宋体、站酷系列、优设标题等。英文部分搭配 DM Serif Display、Outfit、Clash Display 等有特色的英文字体。
- **排版节奏**：中文排版注意行间距（建议 1.7-2.0）、字间距、段落间距的呼吸感。大标题可用负字间距制造紧凑力量感。
- **色彩文化**：注意中文场景色彩的文化含义。金色、红色在中国文化中有特殊含义，可善加利用但避免落入俗套。
- **设计参考**：可参考站酷、优设、腾讯 ISUX、阿里设计等国内优秀设计社区的前沿作品。
